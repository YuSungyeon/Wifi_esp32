# 중간보고서-사용파일

본 문서는 `mac_collector_output/raw/20260427/session_1/device_101.jsonl` 생성 과정에서 실제 사용된 파일(특히 RX/TX 플래시 관련 파일 포함)을 정리한 목록이다.

---

## 1) 최종 산출물(생성된 파일)

### 1. `mac_collector_output/raw/20260427/session_1/device_101.jsonl`
- **역할**: RX(101)에서 전송한 CSI UDP 패킷을 수집기가 라인 단위(JSONL)로 저장한 핵심 결과 파일
- **담고 있는 내용**
  - 수집 시각: `received_at_unix_us`
  - 송신자 정보: `source_ip`, `source_port`
  - 세션/장치/순번: `session_id`, `device_id`, `seq`
  - 무선 메타: `channel`, `rssi_dbm`, `noise_floor_dbm`
  - 신호 본문: `sample_count`, `csi_amp`(길이 52 float 배열)

### 2. `mac_collector_output/raw/20260427/session_1/session_meta_snapshot.yaml`
- **역할**: 수집 시점의 세션 설정을 스냅샷으로 보존
- **담고 있는 내용**
  - 세션 ID, 날짜, 실험 목적
  - 네트워크 설정(SSID/채널/collector 포트)
  - 기대 장치 목록, 운영자 메모

---

## 2) RX 송신(플래시 대상) 관련 파일

### 3. `esp32s3_csi_sender/main/csi_sender_main.c`
- **역할**: RX 펌웨어의 핵심 로직 파일(실제 CSI 수집/전처리/UDP 송신 수행)
- **담고 있는 내용**
  - CSI 콜백 등록 및 활성화(`esp_wifi_set_csi_config`, `esp_wifi_set_csi`)
  - CSI I/Q -> amplitude 변환
  - 1차 전처리(3-tap moving average, z-score, outlier clipping)
  - UDP 패킷 헤더/페이로드 구성 후 수집기로 전송
  - 장치 ID 처리(`DEVICE_ID` 고정 또는 MAC 기반 자동 생성)

### 4. `esp32s3_csi_sender/CMakeLists.txt`
- **역할**: RX 빌드/플래시 시 `-D` 파라미터를 컴파일 매크로로 주입
- **담고 있는 내용**
  - `CSI_WIFI_SSID`, `CSI_WIFI_PASS`
  - `CSI_COLLECTOR_IP`, `CSI_COLLECTOR_PORT`
  - `CSI_SESSION_ID`, `CSI_DEVICE_ID`
  - 위 값들을 `WIFI_SSID`, `COLLECTOR_IP`, `DEVICE_ID` 등으로 컴파일 정의

### 5. `esp32s3_csi_sender/sdkconfig`
- **역할**: ESP-IDF Kconfig 기반의 RX 런타임/빌드 설정 파일
- **담고 있는 내용(핵심)**
  - Wi-Fi CSI 활성화 옵션
    - `CONFIG_ESP_WIFI_CSI_ENABLED=y`
    - `CONFIG_ESP32_WIFI_CSI_ENABLED=y`
  - 콘솔/로그/플래시/RTOS 등 펌웨어 동작 설정 전반

---

## 3) TX/AP 송신(플래시 대상) 관련 파일

### 6. `esp32s3_tx_ap_node/main/tx_ap_main.c`
- **역할**: TX/AP 펌웨어 핵심 로직 파일(SoftAP 생성 + 주기적 heartbeat 트래픽 송신)
- **담고 있는 내용**
  - `WiSLAR_TX_AP` SoftAP 생성
  - 채널 고정(예: 6), 접속 이벤트 로깅
  - `192.168.4.255`로 UDP broadcast heartbeat 송신
  - `seq/timestamp/session_id` 기반 TX 상태 추적

### 7. `esp32s3_tx_ap_node/CMakeLists.txt`
- **역할**: TX/AP 빌드/플래시 설정값 주입
- **담고 있는 내용**
  - `TX_AP_SSID`, `TX_AP_PASS`, `TX_AP_CHANNEL`
  - `TX_AP_BROADCAST_PORT`, `TX_AP_INTERVAL_MS`, `TX_AP_SESSION_ID`
  - 컴파일 시 매크로로 반영되어 코드 수정 없이 운용값 변경 가능

---

## 4) Mac 수집기(저장 생성) 관련 파일

### 8. `mac_collector/udp_collector_mvp.py`
- **역할**: RX UDP 패킷을 수신/검증/저장하는 수집기 엔진
- **담고 있는 내용**
  - 헤더 파싱/검증(`magic/version/header_len/sample_count`)
  - `device_id`별 파일 분리 저장(JSONL)
  - `seq` 누락 기반 `drop_rate` 계산
  - `expected_device_ids`, `missing/stale` 상태 출력
  - `session_meta` 스냅샷 복사 기능

### 9. `mac_collector/UDP_패킷_스키마.md`
- **역할**: ESP 송신 패킷과 Mac 파서의 공통 계약 문서
- **담고 있는 내용**
  - 고정 헤더(40 bytes) 필드 정의
  - payload(`float32 csi_amp[sample_count]`) 규격
  - endian/type/길이 검증 규칙

### 10. `mac_collector/device_registry.csv`
- **역할**: 수집기 expected device 목록의 운영 입력
- **담고 있는 내용**
  - `device_id`, 보드명, MAC, 설치 좌표/높이/방향, 펌웨어 버전, 메모
  - 이번 세션에서는 `device_id=101` 기준으로 수집 검증

### 11. `mac_collector/session_meta.yaml`
- **역할**: 세션 공통 메타데이터 원본
- **담고 있는 내용**
  - `session_id`, 날짜, 실험 목적, 라벨 타깃, split 전략
  - 네트워크/채널/collector 설정
  - expected devices
  - 해당 파일이 수집 시 세션 폴더로 스냅샷 저장됨

## 6) 파일 간 관계 요약

1. TX/AP 펌웨어(`tx_ap_main.c` + `CMakeLists.txt`)가 AP/트래픽 환경을 만든다.  
2. RX 펌웨어(`csi_sender_main.c` + `CMakeLists.txt` + `sdkconfig`)가 CSI를 수집해 UDP 전송한다.  
3. 수집기(`udp_collector_mvp.py`)가 패킷을 스키마 문서(`UDP_패킷_스키마.md`) 기준으로 검증 후 `device_101.jsonl`을 생성한다.  
4. 운영 입력(`device_registry.csv`, `session_meta.yaml`)은 수집기 상태판단 및 세션 메타 스냅샷 생성에 사용된다.

# WiSLAR 배포 단일 매뉴얼

이 문서는 `TX/AP + RX + Mac 수집기 + JSONL->MAT 변환`까지 한 번에 배포/운영하는 기준 절차입니다.

## 1) 대상 구성

- TX/AP 노드 1대: `esp32s3_tx_ap_node`
- RX 노드 2대 이상: `esp32s3_csi_sender`
- Mac 수집기 1대: `mac_collector/udp_collector_mvp.py`

## 2) 사전 준비

### 개발 환경

- ESP-IDF v5.x 설치
- Python 3 환경 준비
- 각 ESP32-S3 USB 연결

### 포트 확인(macOS)

```bash
ls /dev/tty.usbmodem*
```

### 운영 입력 파일 준비

1. `mac_collector/device_registry.csv`
   - `device_id`, `sta_mac`, 위치 정보 업데이트
2. `mac_collector/session_meta.yaml`
   - `session_id`, `network.ssid`, `network.channel`, `network.collector_ip`, `devices.expected_device_ids` 업데이트
3. `docs/실장비_값_입력_체크리스트.md` 전체 점검

## 3) 공통 파라미터 규칙

세션마다 아래 값이 일치해야 합니다.

- TX/AP: `TX_AP_SESSION_ID`
- RX: `CSI_SESSION_ID`
- 세션 메타: `session_meta.yaml`의 `session_id`

네트워크 관련 값도 일치해야 합니다.

- TX/AP: `TX_AP_SSID`, `TX_AP_CHANNEL`
- RX: `CSI_WIFI_SSID`
- 수집기: `--port` / `session_meta.yaml.network.collector_port`

## 4) 실행 순서

### 4-1) Mac 수집기 먼저 실행

프로젝트 루트에서 실행:

```bash
python "mac_collector/udp_collector_mvp.py" \
  --host 0.0.0.0 \
  --port 9999 \
  --output-dir "mac_collector_output" \
  --device-registry-csv "mac_collector/device_registry.csv" \
  --session-meta "mac_collector/session_meta.yaml"
```

확인:

- expected device 목록이 정상 로드되는지 확인
- 초기 `missing_devices`/`stale_devices` 출력 확인

### 4-2) TX/AP 노드 빌드 및 플래시

```bash
cd "esp32s3_tx_ap_node"
idf.py set-target esp32s3
idf.py \
  -DTX_AP_SSID="WiSLAR_TX_AP" \
  -DTX_AP_PASS="wislartx123" \
  -DTX_AP_CHANNEL=6 \
  -DTX_AP_BROADCAST_PORT=3333 \
  -DTX_AP_INTERVAL_MS=20 \
  -DTX_AP_SESSION_ID=1 \
  -DTX_AP_NODE_ID=1 \
  build
idf.py -p /dev/tty.usbmodemTX flash monitor
```

확인:

- `SoftAP started ...`
- `tx heartbeat seq=...` 증가

### 4-3) RX 노드 빌드 및 플래시

중요:

- RX마다 `CSI_DEVICE_ID`를 다르게 사용
- RX마다 `CSI_SESSION_ID`, `CSI_WIFI_SSID`, `CSI_COLLECTOR_IP`는 동일 기준 유지
- 보드 전환 시 캐시 충돌 방지를 위해 `idf.py fullclean` 권장

RX1 예시:

```bash
cd "esp32s3_csi_sender"
idf.py set-target esp32s3
idf.py fullclean
idf.py \
  -DCSI_WIFI_SSID="WiSLAR_TX_AP" \
  -DCSI_WIFI_PASS="wislartx123" \
  -DCSI_COLLECTOR_IP="192.168.4.2" \
  -DCSI_COLLECTOR_PORT=9999 \
  -DCSI_SESSION_ID=1 \
  -DCSI_DEVICE_ID=101 \
  build
idf.py -p /dev/tty.usbmodemRX1 flash monitor
```

RX2 예시:

```bash
cd "esp32s3_csi_sender"
idf.py set-target esp32s3
idf.py fullclean
idf.py \
  -DCSI_WIFI_SSID="WiSLAR_TX_AP" \
  -DCSI_WIFI_PASS="wislartx123" \
  -DCSI_COLLECTOR_IP="192.168.4.2" \
  -DCSI_COLLECTOR_PORT=9999 \
  -DCSI_SESSION_ID=1 \
  -DCSI_DEVICE_ID=102 \
  build
idf.py -p /dev/tty.usbmodemRX2 flash monitor
```

확인:

- 수집기에서 `device=101`, `device=102` packet 증가
- `missing_devices`/`stale_devices` 해소

### 4-4) 데이터 수집

- 최소 5분 정지 베이스라인 수집
- 동작 프로토콜 수집

생성 경로:

- `mac_collector_output/raw/YYYYMMDD/session_<id>/device_<id>.jsonl`
- `mac_collector_output/raw/YYYYMMDD/session_<id>/session_meta_snapshot.yaml`

## 5) 수집 데이터 변환(JSONL -> MAT)

```bash
python "data_tools/jsonl_to_mat.py" \
  --raw-root "mac_collector_output/raw" \
  --output-dir "data" \
  --window-size 192 \
  --stride 96 \
  --labels-csv "data_tools/labels.csv" \
  --session-subject-csv "data_tools/session_subject.csv" \
  --test-subject-ids "S03"
```

결과 확인:

- `data/train_data.mat`
- `data/test_data.mat`

## 6) 운영 중 필수 점검 값

- `missing_devices`: 빈 목록 유지
- `stale_devices`: 빈 목록 유지
- `drop_rate`: 허용 범위 이내 유지
- `session_id`: TX/AP, RX, `session_meta.yaml` 모두 동일

## 7) RX N대 확장 규칙

- 장치 추가 전 `device_registry.csv` 먼저 갱신
- `session_meta.yaml.devices.expected_device_ids` 동기화
- 신규 RX마다 고유 `CSI_DEVICE_ID` 부여
- 고정 ID 정책 예시: `101, 102, 103, ...`

## 8) 자주 발생하는 문제

### 문제 A: RX 데이터가 수집기에 들어오지 않음

- RX `CSI_COLLECTOR_IP`가 Mac IP와 일치하는지 확인
- RX `CSI_WIFI_SSID`와 TX/AP `TX_AP_SSID` 일치 확인
- RX와 TX/AP 채널 일치 확인

### 문제 B: 일부 RX만 수신됨

- `device_registry.csv`의 `device_id`와 RX 빌드값 일치 확인
- `session_meta.yaml.expected_device_ids` 누락 확인
- RX별 USB 포트/플래시 대상 보드 매칭 재확인

### 문제 C: 세션 설정이 섞임

- 세션 시작 전 `session_id`를 하나로 통일
- 장치/환경 변경은 `session_meta.yaml`의 `operator.notes`에 기록

## 9) 배포 체크리스트

- [ ] TX/AP 플래시 완료 및 heartbeat 로그 확인
- [ ] RX 전체 플래시 완료 및 고유 `CSI_DEVICE_ID` 확인
- [ ] 수집기 실행 중 expected device 로드 확인
- [ ] `session_meta.yaml` 최신화 완료
- [ ] 2~5분 테스트 수집에서 `missing/stale` 이상 없음

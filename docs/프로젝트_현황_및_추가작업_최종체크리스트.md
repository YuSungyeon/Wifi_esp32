## 3) 최종 목표 달성을 위한 추가 작업 (우선순위)

## P0 (바로 시작, 필수)

1. **통신/데이터 스키마 확정**
   - RX 노드가 Mac으로 보낼 최소 필드 확정
2. **Mac 수집기 MVP**
   - 디바이스별 수신 + JSONL/Parquet 저장 + seq 유실률 로그
3. **ESP RX 1대 MVP**
   - CSI amplitude 추출 + 간단 필터(이동평균) + UDP 송신
4. **E2E 검증**
   - 5분 정지 데이터 수집 후 프레임율/유실률/기본 분포 체크

## P1 (확장, 강력 권장)

5. **RX 2대로 확장**
   - 동일 세션에서 `device_id`별 동시 수집
6. **라벨링 프로토콜 고정**
   - 동작 클래스 정의, 반복 횟수, 휴식 시간, 위치/방향 규칙
7. **학습용 변환기 작성**
   - raw/proc 데이터를 기존 `.mat` 키 구조로 매핑

## P2 (품질/재현성)

8. **동기화 보정 로직**
   - `timestamp_us` + 수신시각 기반 정렬
9. **실험 자동 리포트**
   - 세션별 수집량, 유실률, 클래스 균형, 신호 통계 자동 출력
10. **기존 학습 코드 정리**
   - 중복 import 제거, 하드코딩 경로/파일명 파라미터화

## 4) 진행 방식 제안 (실행 순서)

1. **데이터 포맷부터 확정**  
   펌웨어/수집기/변환기 모두 이 스키마를 기준으로 개발

2. **단일 RX E2E 먼저 완료**  
   RX 1대로 안정화 후 RX 2대로 확장(디버깅 비용 절감)

3. **학습 파이프라인 연결을 빨리 확인**  
   소량 데이터라도 `.mat` 변환 후 `train.py` 1회 통과를 조기 검증

4. **그 다음에 전처리 고도화**  
   초기에는 amplitude + 경량 필터만 사용하고, 성능 확인 후 feature 확장

## 5) 확정된 의사결정 (사용자 선택 반영)

1. **전송 프로토콜**: UDP 우선
2. **ESP 전송 데이터 형태**: `csi_amp[]` 우선 전송
3. **학습 타깃 라벨**: 이진(`동작/비동작`, `label_mask` 중심)
4. **세션 분할 기준**: 사람 기준 분리
5. **ESP 1차 전처리 범위**: 이동평균 + z-score + 이상치 제거

### 의사결정 반영 메모

- 초기 단계에서는 UDP 기반 빠른 E2E 확보가 우선이며, 유실률이 기준치를 넘으면 TCP 또는 UDP+ACK로 전환합니다.
- `csi_amp[]` 중심 전송으로 구현 복잡도를 낮추고, feature 전송은 P2 이후 확장 항목으로 둡니다.
- 학습 목표를 이진으로 시작해 수집/라벨 품질과 파이프라인 안정성을 먼저 확보합니다.
- 일반화 성능 확보를 위해 사람 기준 분할을 기본 원칙으로 고정합니다.

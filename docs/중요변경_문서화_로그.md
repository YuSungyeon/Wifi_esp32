# 중요 변경 문서화 로그

이 문서는 프로젝트에서 운영/재현성/확장성에 영향을 주는 중요한 변경을 기록합니다.

## 2026-04-25

### RX N대 확장 대응

- `esp32s3_csi_sender/main/csi_sender_main.c`
  - `DEVICE_ID=0` 설정 시 STA MAC 기반 자동 `device_id` 생성 기능 추가
  - 목적: 보드 수 증가 시 수동 ID 충돌 리스크 완화

- `mac_collector/udp_collector_mvp.py`
  - `--expected-device-ids` 기반 missing/stale 모니터링
  - `--device-registry-csv` 추가: 기대 RX ID 자동 로드
  - `--session-meta` 추가: 세션 메타를 수집 결과 디렉터리에 스냅샷 저장
  - 목적: 다수 RX 운영 안정성 + 세션 재현성 확보

### 운영 문서화 체계 추가

- `mac_collector/device_registry_template.csv`
  - `device_id ↔ MAC ↔ 물리 위치` 표준 기록 템플릿
- `mac_collector/session_meta_template.yaml`
  - 세션 조건 표준 기록 템플릿
- `mac_collector/README.md`
  - 등록표 자동 로드/세션 메타 스냅샷 사용법 반영

### 실사용 운영 파일 초기화

- `mac_collector/device_registry.csv` 생성
  - 템플릿 기반으로 초기 RX 장치(101~103) 등록
- `mac_collector/session_meta.yaml` 생성
  - 현재 세션 기본 조건(3m x 3m, UDP, subject-wise, mask) 반영
  - 수집기 실행 시 `--session-meta` 인자로 전달해 세션 스냅샷 저장 가능

### 문서 정합성 점검 및 보완

- `실장비_값_입력_체크리스트.md` 생성
  - 실제 장비값 입력/교차검증/첫 실행 점검 체크리스트 추가
- `프로젝트_현황_및_추가작업_최종체크리스트.md` 업데이트
  - 오늘 구현된 코드/운영 파일 반영
  - 기존 “미구현” 항목을 최신 상태 기준으로 정정

### TX/AP 노드 펌웨어 신규 구현

- `esp32s3_tx_ap_node/main/tx_ap_main.c`
  - SoftAP 생성(SSID/채널/접속자 수 설정)
  - UDP broadcast heartbeat 트래픽 주기 송신 기능 추가
- `esp32s3_tx_ap_node/CMakeLists.txt`
  - 빌드 파라미터(SSID/PASS/채널/송신주기/세션ID 등) 외부 주입 지원
- `TX_AP_노드_적용_가이드.md`
  - 타인이 바로 적용 가능한 상세 플래시/운영/검증 문서 추가

### raw JSONL -> MAT 변환기 신규 구현

- `data_tools/jsonl_to_mat.py`
  - 수집 JSONL을 `train_data.mat`/`test_data.mat`로 변환
  - 기존 학습 코드 호환 키(`*_data_amp`, `*_label_instance`, `*_label_mask`, `*_label_time`) 생성
  - 사람 기준 분할(`session_subject.csv` + `test_subject_ids`) 지원
- `data_tools/labels_template.csv`
  - 라벨 구간 입력 템플릿 추가
- `data_tools/session_subject_template.csv`
  - session-subject 매핑 템플릿 추가
- `data_tools/README.md`
  - 실행 방법/입력 포맷/주의사항 문서화

### 실시간 자세 판정 MVP (2클래스) 구현

- `realtime/realtime_pose_mvp.py`
  - UDP 수신 기반 규칙형 실시간 판정(정지/동작) 추가
  - 장치별 baseline/score/drop_rate 콘솔 출력
- `realtime/README.md`
  - 실행 방법/파라미터 튜닝/교체 계획 문서화
- `data_tools/markers_to_labels.py`
  - 타임라인 마커(`markers.csv`)를 `labels.csv`로 자동 확장 생성
- `data_tools/markers_template.csv`
  - 마커 입력 템플릿 추가

## 문서화 원칙

2. 디바이스 ID 정책 변경

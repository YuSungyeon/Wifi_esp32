# 실장비 값 입력 체크리스트

실험 시작 전에 아래 항목을 순서대로 채우면 바로 수집/검증이 가능합니다.

## 1) 장치 등록표 입력 (`mac_collector/device_registry.csv`)

- [ ] `device_id`를 보드별로 고유하게 지정 (예: 101, 102, 103...)
- [ ] `board_name` 입력 (RX1, RX2, RX3 ...)
- [ ] `sta_mac`를 실제 보드 STA MAC으로 교체
- [ ] `room_x_m`, `room_y_m`를 실제 설치 좌표로 입력 (단위 m)
- [ ] `height_m` 입력 (권장 1.2m 기준)
- [ ] `orientation_deg` 입력 (안테나/보드 방향)
- [ ] `firmware_version` 기록 (예: v0.1.0)
- [ ] `notes`에 설치 위치/특이사항 기록

## 2) 세션 메타 입력 (`mac_collector/session_meta.yaml`)

- [ ] `session_id` 설정 (세션마다 증가)
- [ ] `date`를 실험 날짜로 수정
- [ ] `network.ssid`를 실제 AP SSID로 수정
- [ ] `network.channel`을 실사용 채널로 수정
- [ ] `network.collector_ip`를 MacBook IP로 수정
- [ ] `network.collector_port` 확인 (기본 9999)
- [ ] `experiment.objective`/`label_target`/`split_strategy` 확인
- [ ] `devices.expected_device_ids`를 등록표와 동일하게 맞춤
- [ ] `operator.name`/`operator.notes` 작성

## 3) ESP32-S3 빌드 값 확인 (`esp32s3_csi_sender`)

- [ ] `CSI_WIFI_SSID`가 `session_meta.yaml`의 SSID와 일치
- [ ] `CSI_WIFI_PASS` 확인
- [ ] `CSI_COLLECTOR_IP`가 `session_meta.yaml`의 collector_ip와 일치
- [ ] `CSI_COLLECTOR_PORT`가 `session_meta.yaml`의 collector_port와 일치
- [ ] `CSI_SESSION_ID`가 `session_meta.yaml`의 session_id와 일치
- [ ] `CSI_DEVICE_ID`가 등록표의 해당 보드 `device_id`와 일치
- [ ] 자동 ID 모드(`CSI_DEVICE_ID=0`) 사용 시, MAC 기반 ID를 등록표에 기록

## 4) 실행 전 교차 검증

- [ ] 등록표의 `device_id` 집합 == `session_meta.yaml`의 `expected_device_ids`
- [ ] 수집기 실행 인자에 `--device-registry-csv`와 `--session-meta` 지정
- [ ] ESP 모든 RX가 동일 세션 ID를 사용
- [ ] TX/AP, RX, MacBook이 같은 네트워크 대역에 연결

## 5) 첫 실행 확인 (5분 샘플)

- [ ] 수집기 로그에 `missing_devices`가 빈 목록인지 확인
- [ ] `stale_devices`가 지속적으로 비어있는지 확인
- [ ] `drop_rate`가 허용 범위 이내인지 확인
- [ ] `mac_collector_output/raw/YYYYMMDD/session_<id>/device_<id>.jsonl` 생성 확인
- [ ] 세션 폴더에 `session_meta_snapshot.yaml` 생성 확인

## 6) 실험 종료 후

- [ ] `중요변경_문서화_로그.md`에 구조/운영 변경사항 기록
- [ ] 세션별 이슈(장치 교체, 위치 변경, 간섭 상황)를 `operator.notes`에 반영
